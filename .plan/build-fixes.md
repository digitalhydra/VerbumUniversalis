# Build and Runtime Fixes

This document tracks significant fixes made to the project to resolve build errors or runtime crashes.

## [2024-XX-XX] Room Schema Mismatch Fix

### Issue
The application was crashing on startup with a `java.lang.IllegalStateException: Pre-packaged database has an invalid schema`. 

**Error Log:**
```
java.lang.IllegalStateException: Pre-packaged database has an invalid schema: verses(com.verbum.universalis.data.entities.VerseEntity).
Expected: TableInfo{name='verses', columns={...}, foreignKeys={}, indices={}}
Found: TableInfo{name='verses', columns={...}, foreignKeys={ForeignKey{...}}, indices={Index{...}}}
```

### Root Cause
The `verbum_seed.db` file (generated by `etl/build_verbum_seed.py` and stored in `assets`) contained SQL constraints (Foreign Keys and Indices) that were not declared in the Room `@Entity` classes. Room's `createFromAsset` requires an exact match between the physical database schema and the entity definitions.

### Fix
Updated `app/src/main/java/com/verbum/universalis/data/entities/Entities.kt` to align with the SQL schema defined in the ETL pipeline.

**Changes:**
- **`VerseEntity`**:
    - Added `ForeignKey` to `BookEntity`.
    - Added `Index` named `idx_verses_book` on `(book_id, chapter, verse_number)`.
- **`TextEntity`**:
    - Added `ForeignKey` to `VerseEntity`.
    - Added `Index` named `idx_texts_verse` on `(verse_id, lang_code)`.
- **`InterlinearWordEntity`**:
    - Added `ForeignKey` to `VerseEntity`.
    - Added `Index` named `idx_interlinear_verse` on `(verse_id)`.

### Verification
- Project built successfully using `./gradlew app:assembleDebug`.
- App successfully initialized the database and reached the main dashboard without crashing.

## [2024-XX-XX] Reading Canvas Improvements

### 1. Instant Language Switching
- **Issue**: Changing translation required navigating away and back to see updates.
- **Fix**: Modified `ReadingViewModel` to combine `currentPassage` and `activeLanguage` flows into the `verses` flow. Updated `ReadingCanvas` to use `activeLanguage` as a key for `getDisplayText`, ensuring Compose triggers a re-render immediately upon language change.
- **Files**: `ReadingViewModel.kt`, `ReadingCanvas.kt`.

### 2. Swipe Navigation for Chapters
- **Feature**: Swipe left (right-to-left) for next chapter, swipe right (left-to-right) for previous chapter.
- **Implementation**: 
    - Added `getMaxChapterForBook` to `VerseDao` and `BibleRepository`.
    - Added `nextChapter()` and `previousChapter()` to `ReadingViewModel` with logic for book transitions (Catholic 73-book canon).
    - Implemented swipe detection in `ReadingCanvas` using `Modifier.pointerInput` and `detectDragGestures`.
- **Files**: `Daos.kt`, `BibleRepository.kt`, `ReadingViewModel.kt`, `ReadingCanvas.kt`.
